# 粒子艺术生成器

一个基于 HTML5 Canvas 的交互式粒子艺术生成器，支持多种预设模式和自定义参数调整。

## 功能特性

### 🎨 预设模式
- **星空模式**：繁星点点的夜空效果，粒子缓慢漂移
- **烟花模式**：绚丽的烟花绽放效果，定时发射
- **水流模式**：流动的水粒子效果，带有拖尾

### 🎛️ 可调节参数
- **粒子数量**：50 - 500 个
- **粒子颜色**：自定义颜色选择器
- **粒子速度**：0.1 - 5.0 倍速
- **粒子大小**：1 - 10 像素
- **发光强度**：0 - 30 级辉光效果

### 🖱️ 鼠标交互
- **跟随模式**：粒子被鼠标吸引
- **散开模式**：粒子被鼠标排斥

### 📸 其他功能
- **截图保存**：一键保存当前画布为 PNG 图片
- **重置参数**：快速恢复默认设置
- **响应式布局**：支持不同屏幕尺寸

## 技术栈

- **HTML5 Canvas**：高性能粒子渲染
- **原生 JavaScript**：粒子系统逻辑
- **CSS3**：渐变、发光效果、动画

## 使用方法

### 直接运行
直接在浏览器中打开 `index.html` 文件即可使用。

### 使用本地服务器
推荐使用本地服务器运行以获得更好的体验：

```bash
# Python 3
python -m http.server 8080

# 或 Node.js
npx serve .
```

然后在浏览器中访问 `http://localhost:8080`。

## 核心实现

### Particle 类
单个粒子对象，包含：
- 位置、速度、大小、颜色等属性
- 生命周期管理
- 拖尾效果
- 发光渲染

### ParticleSystem 类
粒子系统管理器，负责：
- 粒子的创建与销毁
- 动画帧循环
- 鼠标交互处理
- 模式切换
- 参数更新

### 性能优化
- 使用 `requestAnimationFrame` 实现流畅动画
- Canvas 半透明覆盖实现拖尾效果
- 对象池复用粒子

## 更新日志

### 2026-06-18 修复优化

#### 1. 烟花模式性能优化（解决卡顿问题）

**问题原因**：
- 烟花粒子数量无上限，持续发射导致粒子不断累加
- 每次发射烟花固定创建30个粒子，高数量设置下粒子总数暴增
- 使用 `splice()` 从数组中删除死亡粒子，大数组下频繁操作性能差

**修复方案**：
- 新增 `maxFireworkParticles = 500` 限制烟花模式最大粒子数
- 动态计算每次烟花发射的粒子数：粒子总数越多，单次发射粒子越少（15-30个）
- 发射前检查粒子数量上限，超过则不发射
- 动态调整发射间隔：粒子数量越多，间隔越长（30-60帧）
- 使用 `aliveParticles` 新数组收集存活粒子，替代反向遍历 + `splice()`，减少数组重排开销
- 烟花粒子拖尾长度从8缩短到6，减少每帧绘制线段数

#### 2. 水流模式交叉射线修复

**问题原因**：
- 水流粒子飞出屏幕底部/两侧后，位置重置到对面
- 拖尾数组中保留了重置前的位置点
- 绘制拖尾时，连接了屏幕两端的点，形成横跨屏幕的长线段（交叉射线）

**修复方案**：
- 粒子重置位置（从底部回到顶部、左右越界）时，清空 `tail` 拖尾数组
- 重置时同时重新初始化速度，避免异常大速度积累
- 绘制拖尾时增加距离检测：相邻两点距离过大（平方距离 > 10000）时，使用 `moveTo()` 替代 `lineTo()`，断开连线

#### 3. 水流模式发光强度修复

**问题原因**：
- 拖尾线条绘制时没有设置 `shadowBlur` 发光效果
- 水流模式粒子较多，拖尾占视觉比重很大，仅粒子圆点发光效果不明显

**修复方案**：
- 绘制拖尾前设置 `ctx.shadowBlur = this.glow * 0.5`（半强度发光）
- 拖尾线条透明度从 0.3 提升到 0.4，线条宽度从 0.5倍 提升到 0.6倍，增强视觉效果
- 粒子本体仍使用完整发光强度，保持层次感
